## 🆕 Последние обновления

### v0.4.0 — Затухание удержания, кристаллизация и оценка качества
- 📉 **Затухание удержания** — Достоверность снижается со временем, если страницы не просматриваются или не подкрепляются. Каждый тип страницы имеет свой период полураспада (концепции: 180 дней, инсайты: 60 дней, черновики: 30 дней). Чтение страницы через `/wiki` сбрасывает таймер.
- 💎 **Кристаллизация** — Когда `/wiki` синтезирует 3+ источника, автоматически создаётся **дайджест-страница** с вопросом, ответом, источниками, ключевыми связями и выводами — превращая ваше исследование в постоянные страницы вики.
- 📊 **Оценка качества** — Каждая страница оценивается от 0 до 100 по 5 измерениям (метаданные, цитирования, свежесть, связи, содержание). Оценки A–F. Страницы ниже 60 отмечаются в `/wiki-update` и `/wiki-audit`.

### v0.3.0 — Аудит вики, фильтр конфиденциальности и разрешение противоречий
- 🔍 **`/wiki-audit`** — Новая команда для проверки здоровья вики: битые ссылки, страницы-сироты, устаревший контент, отсутствующие метаданные, несоответствия цитирований, противоречия, проблемы графа сущностей. Автоматически исправляет что может, остальное сообщает со сводкой оценок качества.
- 🔒 **Фильтр конфиденциальности** — Автоматически удаляет API-ключи, токены, пароли и персональные данные из производного контента при добавлении. Исходные материалы никогда не изменяются. Страницы помечаются `visibility: public/private/internal`.
- ⚖️ **Разрешение противоречий** — Когда источники расходятся, обе стороны оцениваются по давности, авторитетности и количеству доказательств. Более сильные утверждения вытесняют слабые; спорные случаи отмечаются для ручной проверки.

### v0.2.0 — Граф знаний, оценка достоверности и типизированные связи
- 🧠 **Граф знаний** — Каждое добавление извлекает именованные сущности (люди, проекты, технологии, решения) и соединяет их типизированными связями в `.data/entities/`.
- 🔗 **Типизированные связи** — Концепции связываются друг с другом со смыслом: `extends`, `contradicts`, `supersedes`, `depends-on`, `generalizes`, `component-of`.
- 📊 **Оценка достоверности** — Каждая страница имеет уровень `confidence` (`high`/`medium`/`low`), дату `verified`, а краткие изложения отслеживают `authority` источника (`primary`/`secondary`/`commentary`).
- 🔄 **Замещение** — Когда новая информация заменяет старую, страницы связываются через `supersedes`/`superseded_by` — ничего не удаляется, история прослеживается.
- 🕸️ **Запросы с учётом графа** — `/wiki` теперь обходит связи между сущностями наряду с поиском по ключевым словам/семантике, находя связи, которые текстовый поиск пропускает.
- ⚡ **Обнаружение противоречий** — Когда источники не согласны, обе стороны автоматически помечаются и связываются.

> **Обновление с v0.0.x?** Запустите `/wiki-setup` повторно для инициализации нового каталога `.data/entities/`. Существующие страницы вики продолжат работать — новые поля (`confidence`, `verified`, `entities`, типизированные `related`) добавляются постепенно при добавлении новых документов или запуске `/wiki-update`. Миграция не требуется.

### v0.0.x — Основа
- 📅 **Запланированные обновления** — `/wiki-schedule` автоматически запускает `/wiki-update` по расписанию cron через удалённые агенты.
- 🔀 **Смешанное добавление** — `/wiki-ingest` принимает файлы + URL в одной команде.
- 🔄 **Автодобавление при обновлении** — `/wiki-update` обнаруживает и добавляет новые файлы в `raw/`.
- 📝 **Упрощённые команды** — `/wiki-query` переименована в `/wiki`.
- 📁 **Плоская структура** — Удалена обёртка `knowledge-base/`; `raw/` и `wiki/` располагаются в корне проекта.
- 📥 **Пакетное добавление** — Папки, несколько файлов и glob-шаблоны.
- 🔍 **Уровни поиска** — Grep для маленьких проектов, qmd для средних/больших.
- 🔮 **Интеграция с Obsidian** — Автоустановка и настройка во время `/wiki-setup`.
- ⚙️ **Кроссплатформенная автоустановка** — Node.js, qmd, Obsidian на macOS, Linux и Windows.

## 🚀 Быстрый старт

Три шага. Всё устанавливается автоматически — вам не нужно ничего настраивать.

**Шаг 1 — Установить плагин:**
```
/plugin marketplace add bradduy/wiki-knowledge-compiler
```
После добавления настройте его в маркетплейсе:
1. Выполните `/plugin` → выберите вкладку **Marketplace**
2. Найдите **wiki-knowledge-compiler** и откройте
3. Нажмите **Enable** и выберите scope — укажите папку проекта, где будет размещена ваша wiki
4. Подтвердите — этот шаг даёт плагину разрешение на создание файлов и выполнение команд в этой папке

<p align="center">
  <img src="./assets/user_scope_setting.png" alt="Настройка scope" width="100%" />
</p>

**Шаг 2 — Запустить настройку:**
```
/wiki-setup
```
Выберите размер проекта (маленький, средний или большой). Плагин сделает остальное — установит инструменты, создаст папку вики и предложит настроить [Obsidian](https://obsidian.md) для визуального просмотра.

<p align="center">
  <img src="./assets/obisidian.gif" alt="Просмотр wiki в Obsidian" width="100%" />
</p>

**Шаг 3 — Добавить первый документ:**
```
/wiki-ingest ~/Documents/my-article.md
```

Готово. Ваша вики работает.

## 🎯 Использование

Каждая команда подсказывает, что попробовать дальше.

| Команда | Что она делает |
|---------|---------------|
| `/wiki-setup` | ⚙️ Разовая настройка (устанавливает всё за вас) |
| `/wiki-ingest` | 📥 Добавить документы — один файл, целую папку или URL |
| `/wiki` | 🔍 Задать вопрос, получить ответ с источниками |
| `/wiki-insights` | ✨ Найти закономерности и связи между источниками |
| `/wiki-update` | 🔄 Синхронизировать вики (добавить новые файлы из raw/ + обновить индексы) |
| `/wiki-schedule` | 📅 Запланировать автоматический запуск /wiki-update |
| `/wiki-audit` | 🩺 Проверка здоровья вики — исправление битых ссылок, обнаружение противоречий, оценка качества |

**Рекомендуемый порядок:** `/wiki-setup` → `/wiki-ingest` → `/wiki` → `/wiki-insights` → `/wiki-update` → `/wiki-audit` → `/wiki-schedule`

### 📥 Добавление документов

Вы можете добавлять документы разными способами:

```bash
# Один файл
/wiki-ingest ~/Documents/article.md

# Целая папка (сканирует всё внутри)
/wiki-ingest ~/Documents/research/

# Несколько файлов сразу
/wiki-ingest ~/notes/meeting1.md ~/notes/meeting2.md

# Все PDF в папке
/wiki-ingest ~/papers/*.pdf

# URL-адрес
/wiki-ingest https://example.com/interesting-article

# Или просто вставьте текст после команды
```

Для папок и нескольких файлов плагин показывает прогресс обработки.

### 🧪 Пример: От документа к знанию

Допустим, вы добавляете статью об изменении климата:

```
/wiki-ingest ~/Articles/ipcc-summary-2023.md
```

Плагин создаёт:

| Что | Описание |
|-----|----------|
| **Краткое изложение** | Обзор статьи на одну страницу с оценкой достоверности |
| **Концепции** | Страницы для ключевых идей: «углеродный бюджет», «точки невозврата» |
| **Сущности** | Узлы для конкретных вещей — «МГЭИК», «Парижское соглашение», «Д-р Смит» |
| **Тематическая страница** | Страница «наука о климате», связывающая всё воедино |

Добавьте вторую статью об энергетической политике. Плагин обнаружит, что оба источника обсуждают углеродные бюджеты, и автоматически свяжет их — концепции получат типизированные связи (`extends`, `contradicts`, `depends-on`), а сущности соединятся между источниками.

Когда вы спрашиваете:
```
/wiki How do carbon budgets affect energy policy?
```

Плагин не просто ищет по ключевым словам. Он обходит **граф знаний** — начиная с концепции «углеродный бюджет», следуя связям к сущностям и темам — и находит связи, которые поиск по ключевым словам пропустил бы. Вы получаете ответ, основанный на **обоих** источниках, со ссылками на точное происхождение каждого факта.

## 🧠 Граф знаний

Каждый добавленный документ не просто создаёт страницы — он строит **граф знаний**. Плагин извлекает именованные сущности (люди, проекты, технологии, решения) и соединяет их типизированными связями.

```
Redis ──uses──→ Auth Service ──maintained-by──→ Brad
  │                                              │
  └──depends-on──→ PostgreSQL           owns ←───┘
                        │                Auth Migration
                        └──replaces──→ MySQL
```

Когда вы спрашиваете «каково влияние обновления Redis?», плагин идёт от узла Redis по рёбрам `uses`, `depends-on`, `maintained-by` — находя каждый связанный сервис, человека и решение. Не просто страницы, где упоминается «Redis» по имени.

**Что извлекается автоматически:**

| Тип сущности | Примеры |
|-------------|---------|
| Люди | "Brad Duy", "Andrej Karpathy" |
| Проекты | "Auth Migration", "API Redesign" |
| Технологии | "Redis", "PostgreSQL", "Kubernetes" |
| Библиотеки | "React", "PyTorch", "Express" |
| Решения | "Переход на PostgreSQL", "Использование микросервисов" |
| Организации | "Platform Team", "Anthropic" |

## 📊 Достоверность и умное отслеживание

Не все знания одинаково надёжны. Каждая страница вики несёт метаданные:

- **Достоверность** — `high`, `medium` или `low` в зависимости от количества подтверждающих источников
- **Дата проверки** — когда содержание было последний раз подтверждено
- **Авторитетность** — источник `primary` (оригинальное исследование), `secondary` (обзор) или `commentary` (мнение)
- **Типизированные связи** — связи между концепциями имеют значение: `extends`, `contradicts`, `supersedes`, `depends-on`

Когда два источника противоречат друг другу, плагин отмечает **противоречие** и связывает обе стороны. Когда новая информация заменяет старую, старая страница помечается как **superseded** со ссылкой на замену.

## 🔮 Просмотр вики в Obsidian

Во время настройки вас спросят про [Obsidian](https://obsidian.md) — бесплатное приложение для визуального просмотра вики. Оно устанавливается автоматически, если вы согласитесь.

Откройте папку проекта в Obsidian:

- 🕸️ **Граф** — визуальная карта всех страниц и их связей
- 🔙 **Обратные ссылки** — нажмите на страницу, чтобы увидеть всё, что на неё ссылается
- 🔍 **Поиск** — находите что угодно по всей вики
- ⚡ **Синхронизация в реальном времени** — новые страницы от `/wiki-ingest` появляются сразу

Это необязательно. Вики прекрасно работает только с командами.

## 📁 Что создаётся

```
your-project/
  📄 raw/           Ваши оригиналы (никогда не изменяются)
  📚 wiki/          Все сгенерированные страницы (сводки, концепции, темы, инсайты)
  .data/            Внутренние данные (скрыто)
```

Без папки-обёртки. `raw/` и `wiki/` находятся прямо в вашем проекте. Если `raw/` уже существует, настройка просто добавляет `wiki/` рядом.

Все файлы — обычный Markdown. Открывайте любым текстовым редактором, Obsidian или VS Code.

## 💡 Советы

- 🌱 **Начните с малого.** Добавьте 2–3 источника и попробуйте `/wiki`.
- 🎯 **Будьте конкретны.** «Что говорит источник X о Y?» работает лучше расплывчатых вопросов.
- ✨ **Попробуйте `/wiki-insights`** после нескольких источников по теме — он находит закономерности, которые вы можете упустить.
- 🔒 **Ваши источники в безопасности.** Плагин никогда не изменяет оригиналы.

## ⚙️ Как работает настройка

`/wiki-setup` делает всё автоматически в зависимости от размера проекта:

| Размер | Поиск | Что устанавливается |
|--------|-------|---------------------|
| **Маленький** | Встроенный | Ничего дополнительного |
| **Средний** | Умный поиск ([qmd](https://github.com/tobi/qmd)) | Node.js + qmd (автоматически) |
| **Большой** | Самый быстрый | Node.js + qmd + MCP-сервер (автоматически) |

Плагин определяет вашу ОС (macOS, Linux, Windows) и устанавливает всё подходящим способом. При неудаче переключается на встроенный поиск — вы не окажетесь в тупике.

### Клонировать как отдельный проект

```bash
git clone https://github.com/bradduy/wiki-knowledge-compiler.git
cd wiki-knowledge-compiler
```

Откройте в Claude Code и запустите `/wiki-setup`.

### Добавить в существующий проект

```bash
git clone https://github.com/bradduy/wiki-knowledge-compiler.git /tmp/wkc
cp -r /tmp/wkc/.claude/ your-project/.claude/
cp -r /tmp/wkc/.claude-plugin/ your-project/.claude-plugin/
cp -r /tmp/wkc/{agents,skills,templates} your-project/
cp /tmp/wkc/CLAUDE.md your-project/CLAUDE.md
cp /tmp/wkc/.data/wiki.config.md your-project/.data/wiki.config.md
bash /tmp/wkc/scripts/init-kb.sh your-project/knowledge-base
rm -rf /tmp/wkc
```

Затем запустите `/wiki-setup`.

## 📌 Полезно знать

- 📡 **Работает офлайн.** Всё на вашем компьютере.
- 📄 **Простые текстовые файлы.** Не нужна база данных или специальное ПО.
- 📑 **PDF поддерживаются.** Сложные макеты могут извлекаться неидеально.
- 👤 **Для личного использования.** Один пользователь.
- 🕐 **Хотите историю версий?** Выполните `git init` в папке базы знаний.
